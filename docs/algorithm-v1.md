# Алгоритм EdgeFit v1

## Принцип

Алгоритм не пытается выглядеть "умнее, чем надо". Он делает последовательный и объяснимый расчёт:

1. вес задаёт базовый диапазон длины
2. рост слегка корректирует этот диапазон
3. стиль катания и характер доски смещают длину
4. размер ботинка и стойка формируют рекомендацию по ширине
5. каталог фильтруется по длине, весу, стилю, уровню и ширине

## Входные параметры

- `heightCm`
- `weightKg`
- `bootSizeEu`
- `boardLinePreference`
- `skillLevel`
- `ridingStyle`
- `aggressiveness`
- `stanceType`

## Этап 1. Базовая длина по весу

Весовые диапазоны в `engine.ts` дают стартовое окно длины:

- 35-44.9 кг → 138-142 см
- 45-54.9 кг → 142-147 см
- 55-64.9 кг → 147-152 см
- 65-74.9 кг → 152-156 см
- 75-84.9 кг → 155-159 см
- 85-94.9 кг → 158-162 см
- 95+ кг → 161-166 см

## Этап 2. Коррекция по росту

Рост не перебивает вес, а только мягко сдвигает диапазон:

- низкий рост даёт небольшой сдвиг вниз
- высокий рост даёт небольшой сдвиг вверх

## Этап 3. Коррекция по стилю и агрессивности

- `park` смещает длину вниз
- `freeride` смещает длину вверх
- `all-mountain` остаётся базовым сценарием
- `relaxed` немного смягчает рекомендацию
- `aggressive` добавляет стабильности

## Этап 4. Ширина и boot drag

По размеру ботинка формируется ширина:

- до EU 43.5 → `regular`
- от EU 43.5 → `mid-wide`
- от EU 45.5 → `wide`

Стойка влияет только как корректировка риска:

- `duck` даёт чуть больше запаса
- `unknown` заставляет алгоритм быть осторожнее

## Этап 5. Подбор моделей

Для каждой модели оценивается лучший размер по сумме факторов:

- попадание в диапазон длины
- попадание веса в размерную сетку
- совпадение по стилю
- совместимость с уровнем
- достаточная ширина под ботинок
- совместимость с выбранной линейкой досок

## Что важно

- алгоритм детерминированный
- формулы отделены от UI
- правила легко заменить в одном модуле
- тесты фиксируют ожидаемое поведение при будущих изменениях
