# Архитектура MVP

## Цель

Собрать узкий сервис, который быстро доводит пользователя от вопроса "какая доска мне нужна?" до понятного результата и клика в магазин.

## Базовый стек

- `Next.js 16` + App Router
- `React 19`
- `Tailwind CSS v4`
- `PostgreSQL` как основное хранилище
- простая внутренняя админка или CMS на следующем этапе

## Слои проекта

### 1. UI-слой

- `src/app` — маршруты и серверные страницы
- `src/components` — клиентские и презентационные компоненты

UI не должен содержать формулы подбора. Его задача — собрать входные данные, показать результат и перевести пользователя к следующему действию.

### 2. Доменный слой

- `src/lib/quiz/schema.ts` — валидация входных данных квиза
- `src/lib/recommendation/engine.ts` — алгоритм расчёта
- `src/lib/products.ts` — доступ к каталогу моделей
- `src/types/domain.ts` — единый набор доменных типов

Это главный слой, который должен оставаться стабильным при изменении интерфейса.

### 3. Данные

- `db/schema.sql` — целевая SQL-схема для PostgreSQL

На текущем этапе приложение работает только с живым каталогом из базы данных.

## Карта экранов

- `/` — главная
- `/quiz` — пошаговый квиз
- `/result` — экран результата
- `/catalog` — каталог
- `/boards/[slug]` — карточка модели
- `/about`, `/contact`, `/privacy`, `/terms` — служебные страницы

## Поток данных

1. Пользователь проходит квиз.
2. Квиз отправляет данные в `POST /api/recommendation`.
3. API валидирует payload через `zod`.
4. Алгоритм считает длину, ширину, риск и список моделей.
5. Результат сохраняется в `sessionStorage` для MVP-потока.
6. Экран результата читает сохранённый объект и рендерит объяснение.

## Почему это удобно для роста

- можно заменить local/session storage на базу без переписывания UI
- можно менять формулы внутри `engine.ts`, не трогая страницы
- можно подключить реальную аналитику и email capture отдельным слоем
- можно менять способ импорта каталога без перестройки маршрутов

## Следующие инженерные шаги

2. Сохранять `quiz_results` после расчёта.
3. Добавить track-события на квиз и клики по карточкам.
