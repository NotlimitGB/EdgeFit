# Импорт каталога из магазинов

Для массового наполнения каталога добавлен отдельный сценарий:

```bash
npm run import:stores
```

Скрипт:

- забирает текущие модели из `Траектории` через внутренний каталог `/slim/pages/section/snowboard/boards/`;
- забирает текущие модели из `Триал-Спорта` через раздел сноубордов и страницы товаров;
- обновляет цену и ссылку в магазин;
- новые карточки пишет в базу со статусом `draft`;
- уже верифицированные карточки не затирает грубо, а только подтягивает магазинную ссылку и цену, если это безопасно.

## Полезные переменные

Можно ограничивать прогон для проверки:

```bash
STORE_IMPORT_SOURCE=traektoria
STORE_IMPORT_SOURCE=trial
STORE_IMPORT_LIMIT=5
```

Примеры:

```bash
STORE_IMPORT_SOURCE=traektoria npm run import:stores
STORE_IMPORT_SOURCE=trial npm run import:stores
STORE_IMPORT_SOURCE=trial STORE_IMPORT_LIMIT=10 npm run import:stores
```

## Что импортируется автоматически

### Траектория

- бренд и модель;
- цена;
- ссылка в магазин;
- картинка;
- размерная сетка;
- талия;
- диапазон веса райдера;
- жёсткость;
- геометрия;
- стиль катания;
- линейка `men / women / unisex`.

### Триал-Спорт

- бренд и модель;
- цена;
- ссылка в магазин;
- картинка;
- размеры, которые есть на странице товара;
- талия, форма, назначение и жёсткость из файла характеристик.

Важно:

- `Триал-Спорт` не всегда отдаёт рекомендованный вес по ростовкам, поэтому такие размеры сохраняются без достоверного диапазона веса и карточка остаётся `draft`.
- часть товаров `Триал-Спорта` не имеет файла характеристик, поэтому они попадают в предупреждения и пропускаются.

## После импорта

Рекомендуется прогонять:

```bash
npm run lint
npm run test
npm run build
```

И затем смотреть карточки во внутреннем редакторе:

```text
/internal/catalog
```
